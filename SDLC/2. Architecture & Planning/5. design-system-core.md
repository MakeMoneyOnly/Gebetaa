### Token Philosophy
Design tokens are the atomic values that store our design decisions. They create a single source of truth that bridges design and development.

### Token Naming Convention
--[category]-[property]-[variant]-[state]
Examples:
--color-background-primary
--color-text-secondary
--spacing-4
--radius-md
--shadow-lg

### Token Tiers

#### Tier 1: Primitive Tokens (Reference)
Raw values with no semantic meaning - the foundation.

#### Tier 2: Semantic Tokens (Alias)
Contextual meaning applied to primitives - what values mean.

#### Tier 3: Component Tokens
Specific component-level applications - where values are used.
