# SKILL: motion-interactions

## Purpose

Design and implement sophisticated animation systems, micro-interactions, and motion design that elevate the user experience from functional to delightful. This skill creates motion guidelines and implementations worthy of Awwwards and Godly.website recognition.

## When to Use

- After `/component-architecture` defines the UI components
- When adding polish and delight to interfaces
- When implementing page transitions
- When creating loading and state change animations
- When building scroll-driven experiences

### Core Philosophy
Motion should feel **natural**, **purposeful**, and **delightful**. Every animation must serve a function—guiding attention, providing feedback, or creating continuity.

### The 5 Principles of Our Motion System

#### 1. Physical & Natural
Animations should feel like they obey physics. Objects have weight, momentum, and resistance.

```css
/* Bad: Linear, robotic */
transition: all 300ms linear;

/* Good: Natural ease with overshoot */
transition: all 300ms cubic-bezier(0.34, 1.56, 0.64, 1);
2. Purposeful & Meaningful
Every animation must answer: "What is this helping the user understand?"
PurposeExampleJustificationFeedbackButton pressConfirms action registeredContinuityPage transitionShows spatial relationshipGuidanceStaggered revealDirects reading orderDelightSuccess celebrationRewards completion
3. Quick & Responsive
UI animations should be fast. Users should never wait for animations.
Animation TypeDuration RangeMicro-interactions100-200msComponent transitions200-300msPage transitions300-500msComplex reveals500-800ms
4. Consistent & Systematic
Use the same timing and easing across similar interactions.
5. Accessible & Respectful
Always respect prefers-reduced-motion. Provide alternatives for essential information.

// Layout wrapper with page transitions
import { motion, AnimatePresence } from 'framer-motion';
import { useRouter } from 'next/router';

const pageVariants = {
  initial: {
    opacity: 0,
    y: 20,
  },
  enter: {
    opacity: 1,
    y: 0,
    transition: {
      duration: 0.4,
      ease: [0.25, 0.1, 0.25, 1],
      when: "beforeChildren",
      staggerChildren: 0.1,
    },
  },
  exit: {
    opacity: 0,
    y: -10,
    transition: {
      duration: 0.2,
      ease: [0.4, 0, 1, 1],
    },
  },
};

export function PageTransition({ children }) {
  const router = useRouter();
  
  return (
    <AnimatePresence mode="wait">
      <motion.div
        key={router.pathname}
        initial="initial"
        animate="enter"
        exit="exit"
        variants={pageVariants}
      >
        {children}
      </motion.div>
    </AnimatePresence>
  );
}

  const ref = useRef(null);
  const isInView = useInView(ref, { once: true, margin: "-100px" });

      ref={ref}
      initial="hidden"
      animate={isInView ? "visible" : "hidden"}
      variants={revealVariants}
      transition={{ delay }}
    >
      {children}
    </motion.div>
  );
}

import { useScroll, useTransform, motion } from 'framer-motion';

export function ParallaxSection({ children, speed = 0.5 }) {
  const ref = useRef(null);
  const { scrollYProgress } = useScroll({
    target: ref,
    offset: ["start end", "end start"],
  });

  const y = useTransform(
    scrollYProgress,
    [0, 1],
    [`${speed * -100}px`, `${speed * 100}px`]
  );

  return (
    <div ref={ref} className="relative overflow-hidden">
      <motion.div style={{ y }}>
        {children}
      </motion.div>
    </div>
  );
}
