# Motion & Interactions

## 1. Principles

Motion in Gebeta is a functional feature, not just decoration. It serves to guide the user, provide feedback, and create a premium feel.

### 1.1 Core Principles
- **Intentional**: Every animation should have a purpose (e.g., indicating a change in state).
- **Responsive**: Animations must feel snappy (150ms-300ms).
- **Respectful**: Honoring `prefers-reduced-motion` settings is mandatory.

## 2. Animation Library
- **Primary**: `framer-motion`
- **Secondary**: Native CSS transitions for simple hover states.

## 3. Standard Patterns

### 3.1 Page Transitions
- **Enter**: `initial={{ opacity: 0, y: 10 }} animate={{ opacity: 1, y: 0 }}`
- **Duration**: 400ms
- **Easing**: `[0.16, 1, 0.3, 1]` (Custom cubic-bezier for a soft landing)

### 3.2 Micro-interactions
- **Button Active**: `scale: 0.95`
- **Card Hover**: `y: -8, shadow: float-strong`
- **Menu Items (Stagger)**: `staggerChildren: 0.05`

### 3.3 Loading States
- **Skeleton**: Pulsing opacity (0.5 to 1.0).
- **Spinners**: Used only for long-running background tasks.

## 4. Accessibility & Reduced Motion

All motion components must be wrapped or configured to check for user motion preferences.

```tsx
const shouldReduceMotion = useReducedMotion();
// Use static values if true
```

## 5. Interaction Vocabulary
- **Tap**: Primary action trigger.
- **Swipe (Mobile)**: Navigation between categories or dismissing cart items.
- **Long Press**: Secondary actions or context menus (if applicable).
