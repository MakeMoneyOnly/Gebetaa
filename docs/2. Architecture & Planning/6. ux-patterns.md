# UX Patterns

## 1. Core Principles

### 1.1 Mobile-First, Thumb-Friendly
All interactive elements are placed within the natural reach of the user's thumb (bottom-heavy design).

### 1.2 Anticipatory Interface
The app anticipates the user's next move (e.g., suggesting common modifiers for a dish).

## 2. Navigation Patterns

### 2.1 The Global Shell
- **Bottom Navigation**: Primary entry points (Home, Search, Cart, Profile).
- **Dynamic Header**: Context-aware information (Shop Name, Back button).

### 2.2 Category Browsing
- **Horizontal Scroll**: Quick access to categories.
- **Sticky Headers**: Keeping the active category in view during vertical scroll.

## 3. Form Patterns

### 3.1 Input States
- **Default**: Clean, bordered.
- **Focused**: High-contrast border (`brand-crimson`).
- **Invalid**: Red border with helper text.

### 3.2 Feedback Loops
- **Optimistic UI**: Update UI immediately before server confirmation.
- **Toast Notifications**: Minimal interruptions for success/info.

## 4. Error & Loading States

### 4.1 Empty States
Never show a blank page. Every empty state should have:
1. An illustration or icon.
2. A clear title.
3. A relevant Call-to-Action.

### 4.2 Error Boundaries
Graceful recovery from crashes. Log errors silently and show a friendly "Something went wrong" message with a retry button.

## 5. Emerging Market UX

### 5.1 Connectivity Indicators
Notify the user when they are offline but allow them to continue browsing cached data.
- **Label**: "Browsing Offline - Changes will sync later".
- **Visual**: Greyscale or subtle badge.

### 5.2 Performance Budgets
- **First Contentful Paint**: < 1.0s.
- **Interactive**: < 2.5s on 3G.
