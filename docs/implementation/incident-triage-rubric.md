# Incident Triage Workflow and Severity Rubric

## Severity Levels

- Sev1: Critical outage or data exposure affecting core ordering/service operations.
- Sev2: Major degradation of core flows with no workaround.
- Sev3: Partial degradation with workaround available.
- Sev4: Minor defect with low operational impact.

## Initial Response Targets

- Sev1: acknowledge within 5 minutes.
- Sev2: acknowledge within 15 minutes.
- Sev3: acknowledge within 60 minutes.
- Sev4: triage in normal sprint flow.

## Triage Workflow

1. Detect and acknowledge.
2. Classify severity and impacted scope.
3. Assign incident commander.
4. Mitigate customer impact first.
5. Communicate updates at fixed cadence.
6. Resolve, validate, and monitor.
7. Publish post-incident summary with actions.

## Required Incident Record Fields

- Incident ID
- Start time and detection method
- Severity and impacted components
- User/merchant impact
- Root cause summary
- Mitigation and permanent fix
- Follow-up action items with owners and due dates
